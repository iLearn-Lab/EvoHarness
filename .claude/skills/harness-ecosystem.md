# Harness Ecosystem

When the task touches discoverability, workflows, or parity with Claude Code / OpenHarness:

- start with `workspace_status` or `list_registry`
- use `tool_help` for unfamiliar tools
- load skill instructions with the `skill` tool before following them
- inspect commands and agents from the workspace instead of assuming they exist
- check plugins and MCP assets because the user experiences them as part of the harness surface
- if the workspace ecosystem is thin, improve the workspace itself rather than only the examples
