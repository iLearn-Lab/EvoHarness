# Plugin Authoring

- Treat a plugin as a usable bundle, not a folder of markdown leftovers.
- Make the manifest honest about commands, skills, agents, hooks, and MCP entrypoints.
- Keep command names, skill names, and agent roles distinct so discovery feels intentional.
- If the plugin ships MCP, make sure the server can boot from the real workspace and not just from your shell.
- Prefer a few strong bundled workflows over a vague plugin description.
