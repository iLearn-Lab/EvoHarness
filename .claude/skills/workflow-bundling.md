# Workflow Bundling

- Group assets around a user outcome, not around internal implementation layers.
- Prefer one coherent bundle with a clear entrypoint over several tiny disconnected files.
- Check whether the bundle needs a command, a skill, an agent, a plugin, an MCP server, or a combination.
- End with the smallest bundle that still feels complete in live use.
