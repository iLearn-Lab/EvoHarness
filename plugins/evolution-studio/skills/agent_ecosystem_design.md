# Agent Ecosystem Design

- add agents only when they reduce context pressure or sharpen ownership
- every agent should have a bounded job, narrow tool lane, and clear finish line
- prefer reusable agent roles such as review, validation, provider-debugging, and docs scouting
- do not create five variants of the same generic explorer
